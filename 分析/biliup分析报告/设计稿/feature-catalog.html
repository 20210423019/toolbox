📤 3. 上传功能详情

投稿参数

参数类型默认值说明
--titleString空视频标题
--descString空视频简介
--tagString空视频标签（逗号分隔多个）
--tidu32171（直播）投稿分区 ID（200+ 个分区可选）
--coverString空视频封面图片路径
--copyrightu81（自制）1=自制, 2=转载
--sourceString空转载来源 URL
--dynamicString空空间同步动态文字
--dtimei64空（立即发布）定时发布（10位 Unix 时间戳）
--mission-idu64空B站 创作任务 ID
--interactiveu80互动视频标记
--submit枚举client提交接口: client / app / web
-c, --configPath空TOML 配置文件路径（替代命令行参数）

音质与版权

参数默认值说明
--dolby0（关闭）杜比音效: 0=关闭, 1=开启
--hires0（关闭）Hi-Res 无损音频: 0=关闭, 1=开启
--no-reprint0（允许转载）禁止他人转载: 0=允许, 1=禁止
--open-elec0（关闭）开启充电功能: 0=关闭, 1=开启
--up-selection-replyfalse开启精选评论（仅 app 接口）
--up-close-replyfalse关闭评论区（仅 app 接口）
--up-close-danmufalse关闭弹幕（仅 app 接口）
--extra-fields空自定义提交参数（JSON 字符串）

传输选项

参数默认值说明
--line自动探测上传线路: bda2, ws, qn, bldsa, tx, txa, bda, alia
--limit3单文件最大并发分片数
-p, --proxy无HTTP 代理地址
-u, --user-cookiecookies.json登录信息文件路径

⚙️ 6. 技术特性
🔧 核心机制
📦
断点续传 (UploadCheckpoint)
上传中断后自动恢复，跳过已完成文件。checkpoint 按文件路径哈希存储
core
⚡
并发分片上传
单文件分片后并发上传，默认 3 并发，futures::buffer_unordered 实现
core
📊
进度追踪 (Progressbar)
分片级别进度上报，实现 Stream trait，实时反馈
core
🔒
上传互斥锁 (UploadLock)
防止同一账号多进程同时上传，避免限流冲突
core
🏗️ 架构特性
📦
配置文件投稿 (TOML)
支持通过 TOML 配置文件定义投稿参数，适合批量/定时任务
配置
🌐
WebUI 管理服务 (端口 19159)
Next.js 前端 + Rust REST API，图形化管理录制/上传任务
GUI
🔄
多P投稿
一次命令上传多个文件自动合为一个稿件的多个分P
core
📐
av/bv 号自动识别
Vid 枚举统一处理，支持 Av(u64) 和 Bv(String) 互转
API
